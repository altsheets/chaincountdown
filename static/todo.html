v25 = why not also

If we(!) really get to here, something big will have happened.
It might probably be related to... funding ... but not only.

  • multi-threaded http server
    • the current server is not.
    • Ouch, yesterday when I announced this
      • in bitcointalk AND in nxtforum in the same minute ...
      • ... it crashed, and without any error message, it just got stuck. *meh*
    • examples:
      • http://pymotw.com/2/BaseHTTPServer/#threading-and-forking
      • VERY old (2001!) post: http://www.gossamer-threads.com/lists/python/python/62247
      • MultithreadedSimpleHTTPServer https://github.com/Nakiami/MultithreadedSimpleHTTPServer
      • incl. do_POST https://mafayyaz.wordpress.com/2013/02/08/writing-simple-http-server-in-python-with-rest-and-json/
      • google search https://www.google.de/search?q=multi-threaded+BaseHTTPRequestHandler
  • server-app can replace/complement server.py
    • try out heroku, etc.
    • the standalone server is nice, isn't it?
    • but to deploy it in a standard app container - "why not also" !
  • fully templated image text
    • so that all the text lines in the middle
    • can be re-formulated by the user
  • abstract the graphics formatting completely
    • color, font, etc
    • so that it can be used for any other generated image
      • e.g. blocktime measurements, see above
      • e.g. webpage counter, see above
  • user can demand image width and/or height,
    • and all fontsizes are calculated accordingly
  • pull styles from one CSS instead of from many url-query parameters

What do you think about these ideas? Please give feedback in our wikiwiki.
